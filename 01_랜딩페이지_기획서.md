### 03. 공감 (Pain Point)
- 섹션 타이틀: **당일 주문 하나가 이렇게나 많은 문제를 만들어냅니다**
- 리드: "사료가 없다"는 연락 한 통이 그날 생산 일정을 뒤집고, 배차를 바꾸고, 다른 농가 배송까지 밀리게 만듭니다. 매주 반복되고 있지 않으신가요?
- 도미노 4단계(→로 연결):
  1. **긴급 생산 변경** — 이미 짜여진 생산 일정을 당일 급변경해야 합니다.
  2. **배차·루트 재조정** — 기존 배차 순서가 뒤바뀌고, 다른 농가 배송까지 영향을 받습니다.
  3. **추가 배송 비용 발생** — 무계획 긴급 배송은 차량 공회전과 추가 연료비를 만듭니다.
  4. **담당자 업무 폭증** — 잔량 확인 전화·긴급 조정·클레임 대응이 하루 종일 이어집니다.
- 공감 카드 4(아이콘+제목+2줄):
  - **생산 계획 차질** — 사료가 부족해진 뒤에야 주문이 들어옵니다. 확정된 생산 일정을 당일 급변경해야 해 효율이 낮아집니다.
  - **배차 변경 및 배송 혼선** — 긴급 주문에 맞춰 배차 순서가 뒤바뀌면 기존 농가 배송 루트 전체가 영향을 받습니다.
  - **추가 배송 비용 증가** — 계획 없이 나가는 긴급 배송은 공회전·추가 연료비·기사 초과 근무를 만듭니다.
  - **담당자 업무 과부하** — 잔량 확인 전화, 긴급 배차 조정, 클레임 대응이 매일 반복됩니다.
- 전환 메시지: 이 모든 문제의 출발점은 하나입니다. **잔량 데이터 없이 사후 대응만 반복**하기 때문입니다. 미리 파악하고 먼저 대응할 수 있다면, 이 악순환을 끊을 수 있습니다.

### 04. 솔루션
- 타이틀: **잔량 데이터 기반 운영 개선 솔루션**
- 포지셔닝: 단순한 잔량 확인 장비가 아닙니다. 마이피드는 사료빈 잔량 데이터를 기반으로 부족 가능 농가를 미리 파악하고, 당일·긴급 주문을 줄여 배송을 사전에 계획할 수 있게 합니다. 설치 이후에도 데이터 수집 안정성과 운영 개선 효과를 함께 점검합니다.
- 5단계 흐름:
  1. **IoT 장비 설치** — 사료빈 구조·현장을 확인한 뒤 센서를 부착하고 데이터 수집 환경을 세팅합니다.
  2. **잔량 데이터 자동 수집** — 장비가 주기적으로 잔량을 측정해 FMC·Myfeed로 자동 전송합니다.
  3. **부족 가능 농가 선별·알림** — 기준 이하 농가를 자동 표시하고 담당자에게 알림을 보냅니다.
  4. **사전 대응·계획 배송 전환** — 부족 예상 농가에 먼저 연락하거나 기존 배송에 포함해 당일 주문 없이 대응합니다.
  5. **운영 개선·패턴 분석** — 데이터가 쌓일수록 농가별 소비 패턴·주문 주기를 파악해 배송 정확도를 높입니다.

### 05. 핵심 기능 (5 카드)
1. **사료빈 잔량 데이터 자동 수집** — IoT 장비가 잔량을 주기적으로 측정해 시스템에 자동 반영. 직접 확인 없이 실시간 잔량 파악. *(이미지: app_detail)*
2. **농가별 상태 대시보드** — 여러 농가 잔량을 한 화면에서 리스트로 비교, 부족 가능 농가 우선 관리. *(이미지: web_dashboard)*
3. **부족 가능 농가 사전 알림** — 임계치 이하로 떨어지면 자동 알림. 바닥나기 전에 배송 일정 조율. *(이미지: app_alert)*
4. **계획 주문 전환 지원** — 잔량 추이·소비 패턴으로 배송 시점 예측, 당일 주문 비중↓. *(이미지: app_dispatch)*
5. **설치 이후 운영 관리** — 데이터 안정성, 당일 주문 감소 효과, 농가별 개선 여부까지 함께 점검.

### 06. 제품 화면 갤러리
캡션과 함께 실제 스크린샷 배치: 앱 상세(잔량·온도·습도·배터리) / 웹 대시보드(농가별 잔량 그리드) / 소진 예측 알림 / 배차 프리뷰(배송 경유지·배차 확정).

### 07. 기대 효과 (도입 후 기대 효과 · 정성 5)
- **당일·긴급 주문 감소** — 잔량을 사전 파악해 급박한 당일 주문 없이 계획적으로 대응.
- **배차 변경·배송 혼선 감소** — 긴급 주문이 줄어 기존 배차가 안정적으로 유지.
- **전화 확인 업무 절감** — 대시보드로 직접 확인, 반복 전화 불필요.
- **사료 부족 리스크 사전 대응** — 부족 가능 농가를 미리 파악해 클레임·긴급 상황 전 선제 대응.
- **생산·배송 계획 안정화** — 잔량 기반 수요 예측으로 생산·배송 계획을 안정적으로 수립.
- 전환 문구: "감에 의존하는 방식"에서 **"데이터 기반의 사전 관리"**로.

### 08. 현장·설치
- 카피: 실제 농가 사료빈에 센서를 설치하고, 수집된 데이터로 운영을 개선합니다.
- 에셋: install_site(설치 현장), device_fm05c(FM-05C 디바이스)

### 09. 파일럿 안내
- 타이틀: **일부 농가에 먼저 적용해 효과를 직접 검증하세요**
- 카피: 정식 구매 없이, 일부 농가에 먼저 적용합니다. 당일 주문 감소·전화 업무 절감·부족 가능 농가 사전 감지를 데이터로 확인한 뒤, 정식 도입 범위와 비용을 함께 협의합니다.
- 절차 5: 신청 접수 → 현장 상담·대상 농가 선정 → 장비 설치·시스템 연동 → 파일럿 운영·효과 검증 → 정식 도입 범위·비용 협의
- 적합 조직 체크:
  - 당일 주문이 자주 발생해 생산·배차 계획이 흔들리는 사료 회사
  - 농가별 잔량 확인을 전화·수기로 관리하는 조직
  - 배송 계획 변경으로 배송비·운영 부담이 큰 조직
  - 농가별 사료 소비 데이터를 체계적으로 관리하고 싶은 조직
  - 사료 부족 클레임·긴급 대응을 줄이고 싶은 조직
